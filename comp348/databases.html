Point of Clarification

By "database," I am specifically referring to a relational database management system, or RDBMS. The most common type of RDBMS is a SQL database. There are other types of databases out there, like NoSQL (Mongo) and flat-file databases, but we are specifically talking about server-client-type RDBMSes.

Platform-Agnosticism

Much of the guidance provided here is platform-agnostic, meaning that it can generalize to all kinds of SQL databases. Specifically, they all comply with the SQL language definition.

It is important to note that many SQL RDBMSes maintain additional functionality beyond what is specified in the SQL language construct. For example, Oracle SQL features a REGEXP_REPLACE() function that does not exist in other RDBMSes. Your specific flavor of SQL RDBMS may not support the exact functions you need. That means you'll have to achieve that same functionality either programatically in SQL or using an external processor.

Datatypes

Datatypes in SQL-type databases are not the same as datatypes in other languages! They may be similar, though!

  • INT: Integer; some dialects also specify a length for the integer, otherwise it is normally 4 bytes
  • TINYINT: Tiny integer; also commonly used for boolean, as it is only 1 byte, where 0 is FALSE and 1 is TRUE. See below.
  • BIGINT: Big integer; sometimes used in place of REAL
  • DATE: Date, without a time
  • TIME: Time, without a date
  • DATETIME: Date and time
  • TIMESTAMP: A date in relationship to another point in time - avoid if possible
  • CHAR: A single character, similar to C-type languages
  • VARCHAR(): Variable-length character sets (like strings)

Boolean values in SQL

You might notice a conspicuous absence of a boolean type in the SQL specification. In lieu of a proper boolean (true/false) type, most languages use the TINYINT type with two possible values: 0 and 1 (corresponding to FALSE and TRUE, respectively).

A value constraint may be set that allows only the values 0 and 1. If a constraint is not set, then your software is responsible for ensuring that the data are correct.
